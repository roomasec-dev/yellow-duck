# Tool: edr_detections_proxy

用途：
- 通过 console 代理获取检测列表，适合回答"有哪些检测记录""检测的详细信息"。

输入建议：
- 支持分页查询，page 和 page_size 一起规划。
- 如果用户提到第几页、下一页、每页多少条，要把 page 和 page_size 一起规划出来。
- 如果用户查询"该事件关联的风险"，需要传入 incident_id 来筛选关联这个事件的检测记录。
- incident_id 只能来自用户明确提供或真实工具结果里已经出现过的值。

输出建议：
- 优先展示检测 ID、主机名、威胁等级、状态等信息。
- 按最新检测时间优先理解结果。
- 回复里要保留当前页、总页数、是否还有下一页，方便继续翻页。
- 如果结果为空，明确说"当前没有查到检测记录"。
