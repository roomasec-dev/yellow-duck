# Tool: artifact_outline

用途：
- 快速浏览超大 incident/detection 详情的结构，不直接读完整原文。
- 适合在拿到新的 artifact 后，先判断里面有哪些重点对象和值得追的字段。

使用时机：
- 刚拿到较大的 artifact，还不知道应该先搜什么。
- 用户说“先看看这条详情里大概有什么”“先别总结，先摸清结构”。

输出建议：
- 优先关注顶层字段、主要数组、可疑字段族。
- 把高价值字段看成人类调查入口，例如进程、命令行、哈希、文件、注册表、计划任务、服务、网络、时间线。
- outline 之后通常要继续规划 artifact_search；只有用户明确只想看概览时，才停在 outline。
