# Tool: edr_batch_deal_incident

用途：
- 批量处置事件（标记为已处理/误报等），属于高危写操作。

风险要求：
- 必须等待用户明确回复"确认"后再执行。
- 执行前要确认事件 ID 和处置状态明确无歧义。

输入说明：
- ids：事件 ID，多个用逗号分隔，必填。
- allow：是否标记为误报，必填，true-标记为误报 false-标记为已处理。
- status：处置状态，必填, 1-未处置 2-处置中 3-已处置 4-反馈误报。
- scene：场景，必填，枚举值 batch(批量处置) 或 alone(单独处置)，默认 batch。
- comment：备注，可选。

状态映射表：

| 用户说法 | allow | status |
|---------|-------|--------|
| 误报、这是误报、false positive | true | 4 |
| 已处置、已处理、resolved | false | 3 |
| 处置中 | false | 2 |
| 未处置 | false | 1 |

输出建议：
- 成功后汇报批量处置的事件数量。
