# Tool: scheduled_task_feedback

用途：
- 把用户对定时任务结果的反馈写回任务记忆，用于后续去重、抑制重复告警、保留人工研判结论。

适用场景：
- “这个是误报”
- “这个已经处理了”
- “别再报这个”
- “这个继续关注”

输入建议：
- `task_feedback` 建议使用：`false_positive`、`resolved`、`watch`
- 如果有 `task_id` 就带上；没有时可以默认作用于最近一轮定时任务结果。

规则：
- 如果用户说“这个”，默认理解为最近一轮定时任务汇报里提到的对象。
- 写回后，子agent下次执行要带着这份记忆，不要重复推送已研判对象。
