# Tool: edr_tasks

用途：
- 查询指令任务列表，或者叫人工响应列表，适合回答"有哪些指令下发了""人工响应任务执行状态怎么样""某个主机的指令任务情况"。

输入建议：
- 支持分页查询，page 和 page_size 一起规划。
- 如果用户提到第几页、下一页、每页多少条，要把 page 和 page_size 一起规划出来。
- 如果用户只说"下一页"，优先根据最近一次任务列表的页码继续翻页。

输出建议：
- 优先展示任务 ID、主机名、指令名称、状态（0-下发中 1-执行中 2-已完成 3-失败 4-超时）、操作用户。
- 按最新任务时间优先理解结果。
- 重点关注状态为"失败"或"超时"的任务，这些可能需要进一步处理。
- 回复里要保留当前页、总页数、是否还有下一页，方便继续翻页。
- 如果结果为空，明确说"当前没有查到指令任务"。
