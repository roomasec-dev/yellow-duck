# Tool: edr_incidents

用途：
- 查询近期安全事件，适合回答“最近有什么告警/事件”“某台主机最近有没有事件”。

输入建议：
- 用户提到 client_id 时优先按 client_id 查。
- 没有明确主机时，可以先查全局近期事件。
- 如果用户提到第几页、page、下一页、每页多少条，要把 page 和 page_size 一起规划出来。
- 如果用户只说“下一页”，优先根据最近一次 incidents 列表的页码继续翻页。

ID 约束：
- 如果后续可能继续查看某条 incident 详情，必须优先保留并复用列表结果里的真实 `incident_id` 和 `client_id`。
- 不要把 `host_name`、`incident_name`、主机别名、时间片段，或自己拼出来的字符串，当成 `incident_id`。
- 如果当前上下文里没有真实 `incident_id`，就先停在列表层，或明确告诉用户需要先从列表里选一条。

输出建议：
- 优先总结事件数量、最高风险、关联主机。
- 默认按最新时间优先理解结果，先看最近发生的 incident。
- 不要只是机械复述字段，要告诉用户这些事件意味着什么。
- 回复里要保留当前页、总页数、是否还有下一页，方便继续翻页。
- 如果用户说“随机看一个详情”“挑一个看看”“你帮我找一个典型的”，优先从真实列表里选一条信息更完整、风险更高或行为链更清晰的 incident，然后继续调用详情链路，不要停在列表摘要。
- 进入详情链路后，最终回答不要只报 ID 和字段，要把事件经过、涉及主机、关键行为、时间线和为什么值得关注讲清楚。
