# Tool: artifact_read

用途：
- 读取超大 incident/detection 详情里的连续片段。
- 适合在搜索命中后，围绕某段上下文继续展开看。

使用时机：
- 已经知道大致位置，或者需要看某段上下文。

输入建议：
- start_line 尽量基于已有搜索命中位置。
- line_count 不要太大，优先小块阅读。
- 如果用户没给 artifact_id，默认优先用最近一次 artifact。

输出建议：
- 返回带行号的片段。
- 只读必要范围，不要把整份大对象一次性倾倒出去。
- 如果当前片段还不足以讲清楚事件经过，应该继续搜索或再读相邻片段，而不是过早结束在“我读到了这些行”。
- 如果当前片段已经能支撑用户问题，就直接基于真实片段完成回答。
- 单个高危事件深挖不要追求把整份详情读完；拿到能解释原因的关键证据后就该总结。
- 面向用户转述时保持纯文本，不要把解释写成 markdown 报告。
- 面向用户转述时不要泄露内部状态，不要出现“辅助分析”“根据刚才的搜索结果”“enough_to_answer=true”“next_queries”这类内部术语。
