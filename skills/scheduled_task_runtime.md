# Capability: scheduled_task_runtime

目标：
- 让定时任务成为真正可用的 agent，而不是每轮都失忆。

执行原则：
- 子agent每次执行时都要带上任务自己的记忆，不是从零开始。
- 已经汇报过的对象，默认不要重复推送。
- 用户明确说“误报”“已处理”后，后续同一任务再扫到同一对象时，默认压制。
- 只有新对象，或者同一对象有明显变化时，才再次汇报。

汇报原则：
- 定时任务结果默认直接回到聊天，不要每轮都问用户下一步。
- 如果本轮没有新变化，可以静默，不必刷屏。
- 如果要汇报，先给一句结论，再列新对象及主机、client_id、关键线索。
