# Tool: artifact_search

用途：
- 在超大的 incident/detection 详情里按关键词定向搜索。
- 适合回答“这条详情里有没有某个进程/文件/注册表/命令行/用户/主机字段”。

使用时机：
- 当上一轮已经生成了大对象 artifact。
- 用户在追问刚才那条大详情里的某个局部线索。

输入建议：
- query 要尽量贴近用户问题里的关键词。
- 如果用户没给 artifact_id，默认优先用最近一次 artifact。

输出建议：
- 返回命中的行号和局部片段。
- 如果没搜到，明确说没命中，不要脑补。
- 如果搜索目标是 incident/detection 详情里的关键行为链，命中后应继续规划 `artifact_read` 去读上下文，再把事件经过讲清楚，不要只停留在搜索命中列表。
- 但如果当前辅助分析已经足够回答用户问题，就不要继续搜索；应直接结束工具链并回答用户。
- 最终面向用户转述时，不要暴露内部状态，不要出现“根据刚才的搜索结果”“enough_to_answer=true”“next_queries”这类内部术语。
