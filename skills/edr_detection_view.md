# Tool: edr_detection_view

用途：
- 查询单条 detection/告警的详情。
- 适合回答“这条告警具体命中了什么”“某个 detection_id 的 process 视图内容是什么”。

输入要求：
- 必须有 detection_id。
- 必须有 client_id。
- 如果用户给了 view_type 和 process_uuid，也要一并带上。
- `detection_id` 只能来自用户明确输入或真实工具返回，不能靠 host、进程名、时间、描述文本自行猜测。
- 如果上下文里没有真实 `detection_id`，先停在列表或让用户确认目标，不要硬调详情。

输出建议：
- 默认用纯文本自然说明，不要用 markdown 标题、加粗、代码块、表格，也尽量不要写机械编号。
- 优先说明这条检出的主体、视图类型和关键证据。
- 默认补一个简单进程链，说明是谁触发了谁、后面有没有注入、落地、联网或持久化动作。
- 默认补一个“事件 IOC”小段，只列真实出现过的文件名、路径、哈希、进程名、命令行、IP、域名、注册表项、计划任务、服务名、自启动项。
- 如果用户只是要原始内容，可以直接给原始详情字段。
- 如果需要分点，也优先用短横线或自然分段，不要写成 markdown 报告体。
- 不要向用户暴露内部分析状态，不要出现“辅助分析”“根据刚才的搜索结果”“enough_to_answer=true”“next_queries”这类内部术语。
